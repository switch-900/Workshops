Workshop 1: Development Environment Setup

📋 Prerequisites

Before we begin, ensure you have administrative access to your computer and a stable internet connection.

🖥️ Select Your Operating System

Choose your operating system for customized instructions:

1

Install Visual Studio Code

Download and install the latest version of Visual Studio Code for your operating system.

Download: code.visualstudio.com

Windows: Download the "System Installer" (.exe file)

1. Run the downloaded .exe file 2. Accept the license agreement 3. Check "Add to PATH" option 4. Check "Register Code as an editor for supported file types" 5. Complete the installation

macOS: Download the Universal .dmg file

1. Open the downloaded .dmg file 2. Drag Visual Studio Code to Applications folder 3. Launch from Applications or Spotlight 4. Allow security permissions if prompted

Linux: Multiple installation options available

# Debian/Ubuntu (recommended): wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg sudo install -o root -g root -m 644 packages.microsoft.gpg /etc/apt/trusted.gpg.d/ echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/trusted.gpg.d/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" | sudo tee /etc/apt/sources.list.d/vscode.list sudo apt update sudo apt install code # Or download .deb package from website
2

Create GitHub Account & Sign In

GitHub will be our central platform for version control and collaboration. If you don't have an account, create one first.

Sign up: github.com

Once you have an account, sign in to GitHub through VS Code:

1. Open VS Code 2. Click the Account icon (bottom left) 3. Select "Sign in to sync settings" 4. Choose "Sign in with GitHub" 5. Authorize VS Code in your browser
3

Install GitHub Copilot

GitHub Copilot is an AI-powered code completion tool that will accelerate your development process.

Installation steps:

1. Open VS Code Extensions panel (Ctrl+Shift+X) 2. Search for "GitHub Copilot" 3. Click "Install" on the official GitHub Copilot extension 4. Sign in when prompted 5. Follow the activation prompts

Note: GitHub Copilot requires a subscription, but offers a free trial

4

Install Essential Extensions

Let's set up some essential extensions that will enhance your development experience:

• Live Server - Real-time preview of web pages • GitLens - Enhanced Git capabilities • Prettier - Code formatter • Auto Rename Tag - Automatically rename paired HTML tags • Bracket Pair Colorizer - Color-coded bracket matching

Install each by searching in the Extensions panel and clicking "Install".

Next, configure a VS Code task so you can launch Live Server from the Tasks menu:

// create a file at .vscode/tasks.json { "version": "2.0.0", "tasks": [ { "label": "Live Server", "type": "shell", "command": "live-server --port=5500", "group": "build", "presentation": { "reveal": "always" }, "problemMatcher": [] } ] }
5

Configure Git in VS Code

Set up Git with your credentials using VS Code's built-in interface:

Method 1 - Using VS Code Settings: 1. Open VS Code Settings (File → Preferences → Settings) 2. Search for "git.user" 3. Set "Git: Default Clone Directory" to your preferred folder 4. Set your Git user name and email in the Git section

Windows: If Git isn't installed, VS Code will prompt you to install it

1. Download Git from git-scm.com 2. Install with default options 3. Restart VS Code 4. Open Command Palette (Ctrl+Shift+P) 5. Type "Git: Initialize Repository"

macOS: Git comes pre-installed, but you might want the latest version

1. Install Xcode Command Line Tools if prompted 2. Or install via Homebrew: brew install git 3. Open Command Palette (Cmd+Shift+P) 4. Type "Git: Initialize Repository"

Linux: Install Git via your package manager

# Ubuntu/Debian: sudo apt update && sudo apt install git # Fedora: sudo dnf install git # Arch: sudo pacman -S git

Configure using VS Code's Source Control panel:

1. Click Source Control icon in sidebar (Ctrl+Shift+G) 2. When prompted, enter your Git username and email 3. Or use Command Palette → "Git: Configure"
6

Create Your First Repository

Let's create a repository for our workshop projects:

1. Go to github.com 2. Click "New" (green button) 3. Name it "switch-900-workshops" 4. Add description: "Projects from Switch 900 workshop series" 5. Make it Public 6. Initialize with README 7. Click "Create repository"
7

Clone Repository Using VS Code

Use VS Code's built-in Git interface to clone your repository:

Method 1 - Command Palette (Recommended): 1. Open VS Code 2. Press Ctrl+Shift+P (Cmd+Shift+P on Mac) 3. Type "Git: Clone" 4. Paste your repository URL from GitHub 5. Choose a local folder to save the project 6. Click "Open" when prompted
Method 2 - Welcome Screen: 1. Open VS Code 2. Click "Clone Git Repository" on Welcome screen 3. Paste repository URL 4. Select destination folder 5. Open the cloned repository
Method 3 - Source Control Panel: 1. Click Source Control icon (Ctrl+Shift+G) 2. Click "Clone Repository" button 3. Enter repository URL 4. Choose folder location

Tip: VS Code will automatically detect it's a Git repository and show version control features

8

Verify Setup with VS Code Git Interface

Let's test everything using VS Code's visual Git tools:

1. Create a new file: workshop1-test.html 2. Add some basic HTML (let Copilot help!) 3. Install Live Server extension if not done yet 4. Right-click the file → "Open with Live Server"

Commit changes using VS Code:

1. Click Source Control icon in sidebar (Ctrl+Shift+G) 2. You'll see your new file under "Changes" 3. Click the "+" next to the file to stage it 4. Type a commit message like "Add workshop 1 test file" 5. Click the checkmark (✓) to commit

Push to GitHub using VS Code:

1. In Source Control panel, click "..." (three dots) 2. Select "Push" from the menu 3. Or click the sync icon in the status bar 4. Authenticate with GitHub if prompted

Success! Check GitHub.com - your changes should appear there. Your development environment is ready!

Workshop 2: Node.js Setup & Ordinal Template

📋 Prerequisites

Ensure you have completed Workshop 1 and have Visual Studio Code with GitHub and Copilot set up.

1

Install Node.js

Node.js is a JavaScript runtime that allows us to run JavaScript outside the browser and manage project dependencies.

Download: nodejs.org/en/download

Windows: Download the LTS (Long Term Support) version

1. Download the Windows Installer (.msi) 2. Run the installer as Administrator 3. Accept the license agreement 4. Keep default installation path 5. Ensure "Add to PATH" is checked 6. Complete the installation 7. Restart VS Code after installation

macOS: Download the macOS Installer

1. Download the macOS Installer (.pkg) 2. Double-click to run the installer 3. Follow the installation wizard 4. Enter your admin password when prompted 5. Complete the installation 6. Restart Terminal and VS Code

Linux: Install via package manager or download binary

# Ubuntu/Debian: curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - sudo apt-get install -y nodejs # Or using snap: sudo snap install node --classic # Verify installation: node --version npm --version

Verify Installation:

1. Open VS Code Terminal (View → Terminal) 2. Type: node --version 3. Type: npm --version 4. Both should return version numbers
2

Clone the Ordinal Template Repository

We'll use @boppleton's vite-ordinal-template, which is specifically designed for creating Bitcoin ordinal inscriptions.

Using VS Code's Git Interface:

1. Open VS Code 2. Click the Source Control icon in the sidebar (Git extension) 3. Click "Clone Repository" button 4. In the search bar, paste: https://github.com/ordengine/vite-ordinal-template.git 5. Choose a folder to save the project (create "Ordinal-Projects" folder) 6. Click "Open" when cloning is complete

Alternative method using Command Palette:

1. Press Ctrl+Shift+P (Cmd+Shift+P on Mac) 2. Type "Git: Clone" 3. Paste: https://github.com/ordengine/vite-ordinal-template.git 4. Select destination folder 5. Open the cloned repository

Important: This template is specifically designed for Bitcoin ordinals and maintains compatibility with on-chain inscriptions.

3

Install Project Dependencies

Instead of the standard npm install, we'll use a special flag to maintain compatibility with the specific versions used in on-chain inscriptions.

1. Open VS Code Terminal (View → Terminal or Ctrl+`) 2. Make sure you're in the project directory 3. Run: npm install --legacy-peer-deps 4. Wait for installation to complete (may take a few minutes)

Why --legacy-peer-deps?

• Ensures compatibility with specific package versions • Maintains alignment with inscriptions already on-chain • Prevents version conflicts in the ordinal ecosystem • Required for this particular template setup

Note: You might see some warnings during installation - this is normal for this template.

4

Run the Development Server

There are two ways to view and work with your code:

Method 1 - Development Mode (Recommended for coding):

1. In VS Code Terminal, run: npm run dev 2. Wait for the server to start 3. Look for output like: "Local: http://localhost:5173" 4. Click the link or copy/paste into your browser 5. Your ordinal template will open in the browser

Development Mode Benefits:

• Live reloading - changes appear instantly • Hot module replacement for faster development • Better error messages and debugging • Perfect for testing and iterating on your code

Keep the terminal open: The dev server runs continuously. To stop it, press Ctrl+C in the terminal.

5

Build for Production

When you're ready to create the final version for inscription, you'll build the production version.

1. Stop the dev server (Ctrl+C if running) 2. In terminal, run: npm run build 3. Wait for build process to complete 4. A new "build" folder will be created 5. Inside build, you'll find index.html (your final inscription file)

Why build periodically?

• Ensures no conflicts in your code • Creates optimized, minified version • Tests production compatibility • Generates the exact file you'll inscribe • Catches build errors early

Best Practice: Build regularly during development to catch issues early!

6

Test in Ordinals Environment

Before inscribing, always test your code in the ordinals inscription tester to ensure it works in the Bitcoin environment.

1. Run npm run build to create the production version 2. Open build/index.html in VS Code 3. Select all the code (Ctrl+A) and copy it (Ctrl+C) 4. Go to: https://ordinals.com/content/4cc26939b8c375b75d283c44c1c1a02f9b28a33417d233a9b8fccbc4482aa102i0 5. Paste your code into the inscription tester 6. Your ordinal will run in the Bitcoin ordinals environment

What the inscription tester does:

• Simulates the Bitcoin ordinals environment • Tests compatibility with ordinal protocols • Ensures your code works without external dependencies • Validates your inscription before actual deployment

Success! If your code runs in the tester, it's ready for inscription on Bitcoin!
